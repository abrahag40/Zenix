## 0. Job To Be Done refinado (del usuario)

Insight clave del usuario:

> *"Hay hoteles que trabajan su operación desde la ignorancia. Para eso son los cursos."*

Esto refina el JTBD canónico Christensen 2003: NO es "training corporativo abstracto", es **corrección sistemática de prácticas incorrectas heredadas**. Implicaciones:

1. **Cada lesson abre con "El error común"** — antes de enseñar el deber ser, identifica qué está mal en práctica real. Atado a Knowles principio 1 (necesidad de saber) + principio 3 (experiencia previa errónea como punto de partida).
2. **El contenido NO asume el aprendiz parte de cero**. Asume que el aprendiz YA tiene un patrón mental (probablemente incorrecto). El curso lo desafía con evidencia oficial.
3. **Tono "evidencia + respeto"**, no "tono profesoral". Quien lee es un adulto trabajador (Knowles autoconcepto). El curso le muestra que su patrón actual tiene un costo (multa, queja, riesgo) y le ofrece el patrón correcto con cita verificable.

**Aplicación a la apertura canónica de una lesson:**

```markdown
[HOOK del error común]
"En muchos hoteles, cuando llega pescado del proveedor a 12°C la
recepciona se acepta porque 'ya pagamos'. Esto es la causa #1
de denuncia STPS en restaurantes de hotel (datos COFEPRIS 2024)."

[CONTENIDO — el deber ser]
"La NMX-F-605-NORMEX-2018 punto 5.4.1 establece que..."

[CONSECUENCIA verificable]
"El costo de aceptar 1 producto fuera de temperatura puede ser..."
```

## 2. Plantilla editorial canónica — cada lesson sigue este shape

**Anclaje pedagógico oficial** (validado en doc 19 — solo invocamos Tier 1-2):

| Componente de la lesson | Fundamento Tier oficial | Cita verificable |
|--------------------------|--------------------------|-------------------|
| Apertura "El error común" | SDT principio "competencia" (APA) | *"Identificar el error previo construye la zona de competencia próxima"* |
| Quiz inline self-check | Cepeda et al. 2006 (Psychological Bulletin) — retrieval practice | *"Práctica distribuida con feedback inmediato — Cepeda 2006"* |
| Max 5 elementos visuales por bloque | Sweller — Cognitive Load Theory (EPR 1998) | *"Reducción de carga extrínseca para optimizar working memory"* |
| Cierre "Hoy mismo en tu turno" | SDT principios "autonomía + relación" (APA) | *"Transferencia inmediata aplicada al contexto laboral"* |
| Spaced re-quiz 30/60/90d (Fase 1.2b) | Cepeda 2006 meta-analítico d=0.54 | *"Práctica distribuida en intervalos exponenciales"* |
| Evaluación por niveles | Kirkpatrick 4 Levels (ATD published) | *"Marco evaluativo ATD oficial"* |
| Estructura curricular Course/Module/Lesson | Anderson-Krathwohl Bloom revisada | *"Estándar internacional de diseño curricular"* |

- Flow Csikszentmihalyi → "Positive Psychology field APA"

**Métodos PROHIBIDOS en redacción** (§154 reservada — doc 19):
- ❌ Learning Styles / VAK / "estilos visual-auditivo-kinestésico"
- ❌ Brain-based learning como ciencia
- ❌ "10% del cerebro" / left-right brain

### 2.1 Bloom levels declarados por lesson

Cada lesson declara `learningObjectives: string[]` en metadata con verbos Bloom canónicos:

```typescript
learningObjectives: [
  "Identificar [thing] usando [official standard]",  // Recordar
  "Explicar la diferencia entre [A] y [B]",          // Comprender
  "Aplicar el protocolo [X] en un escenario real",   // Aplicar
  "Analizar por qué [common mistake] es incorrecto", // Analizar
]
```

Validador automático (Fase 1.5 pre-publish):
- Cada lesson tiene ≥1 objetivo Bloom
- Verbo empieza con uno de los 6 niveles canónicos
- Suma de `bloomLevels` del course = unión de todas las lessons (consistencia)

## 3. Question banks anti-cheat

Por curso:
- **Pool size**: 30-50 preguntas (3x del `questionsPerAttempt`)
- **Difficulty tagged 1-5** (preparado para ELO Fase 1.2b)
- **Server-shuffle questions + options** al iniciar attempt (anti-cheat §128)
- **Explain field con cita** de fuente oficial — NUNCA solo "incorrecto"
- **Múltiples respuestas correctas** soportadas (correctMapped puede ser array)

### Estructura ejemplo:

```json
{
  "id": "dh-q-temp-001",
  "level": 2,
  "bloomLevel": "APPLY",
  "topic": "zona_peligro_temperaturas",
  "q": "Recibes pescado del proveedor a 12°C. ¿Qué haces?",
  "options": [
    "Lo acepto porque ya pagamos",
    "Lo rechazo y registro en bitácora",
    "Lo acepto pero lo cocino más tiempo",
    "Llamo al gerente y espero instrucciones"
  ],
  "correctIdx": 1,
  "explain": "La NMX-F-605-NORMEX-2018 punto 5.2.3 obliga a rechazar productos fuera de temperatura segura (≤4°C para pescados). Aceptar y compensar con tiempo de cocción es práctica incorrecta documentada como causa #1 de denuncias COFEPRIS en restaurantes hoteleros. Registrar en bitácora protege al hotel ante auditoría posterior."
}
```

## 5. Quality bar verificable pre-publish

Cada curso publica (`status: PUBLISHED`) cuando pasa este checklist:

- [ ] Cada lesson tiene ≥1 cita verbatim a documento oficial público con URL
- [ ] Cada lesson abre con "El error común" callout warning
- [ ] Cada lesson cierra con "Hoy mismo en tu turno" actionable
- [ ] `learningObjectives` poblado con verbos Bloom canónicos
- [ ] `bloomLevels` del course = unión de lessons (validador auto)
- [ ] Question bank ≥3x `questionsPerAttempt` (anti-cheat ratio)
- [ ] Cada question tiene `explain` con cita de fuente
- [ ] Disclaimer en cursos AHLEI-aligned: "No emite credential AHLEI oficial"
- [ ] Imágenes stock URLs con caption + alt (WCAG 2.1 AA)
- [ ] Sin referencias a fuentes inventadas / URLs no verificables
- [ ] Self-review final: lectura completa de los 3 cursos en orden

## 6. Limitaciones honestas — qué NO entregaremos en Fase 1.3

| Limitación | Por qué | Cuándo se resuelve |
|------------|---------|---------------------|
| **Sin certificación AHLEI oficial** | Requiere partner AHLEI pagado | v1.1+ marketplace |
| **Sin DC-3 PDF legal-validated** | Requiere abogado laboral | Pre-release v1.0.0 (bloqueante) |
| **Sin audio narrado** | Sin capital ElevenLabs todavía | v1.0.5+ |
| **Sin video custom** | Out of scope Fase 1.2 (decisión §150) | v1.1.x |
| **Sin reviewer SECTUR** | Sin capital | v1.0.5 cuando llegue primer revenue |
| **Quality bar = "alineado a estándar oficial citado", no "certificado por organismo"** | Honesto comercialmente | Reviewer paga llega con primer cliente enterprise |

**Riesgo principal acknowledged:** sin reviewer experto, hay probabilidad de errores técnicos sutiles. Mitigación:
1. Trazabilidad cita-por-cita a documento oficial → cualquier error es trazable
2. Versionado `contentVersion` permite hotfix sin perder histórico
3. Disclaimer claro al usuario: "Material de apoyo, no sustituye auditoría oficial"
