## 0. Principio editorial fundacional (LEER ANTES DE REDACTAR)

**Este curso NO es sobre cómo usar el PMS Zenix.** Es sobre **mejores prácticas de recepción hotelera alineadas a estándares globales** (AHLEI CFDR, CONOCER EC0124, PCI-DSS, Visa Core Rules, Kotler). El objetivo es **agregar valor al hotel a través de su personal de recepción** — usando estándares portables que el staff conserva incluso si cambia de hotel o de PMS.

**Cuando se hable de tecnología (Módulo 2):** se enseña qué ES un PMS, un channel manager, un GDS en **términos genéricos de la industria** — NO cómo operar Zenix específicamente. Ejemplos genéricos: "el calendario de reservas", "el estado de habitación (vacante/ocupada/sucia/limpia)", "el folio del huésped". NUNCA términos Zenix-internos (daily grid, SSE task:ready, batchCheckout, etc.).

El "Curso Zenix Sistema" (cómo operar el PMS Zenix por rol) es producto **separado**, gratuito vía Activate wizard, producido en fase posterior.

**Disclaimer obligatorio en cada lesson:** *"Curso ALINEADO al estándar AHLEI CFDR. NO emite certificación AHLEI oficial — requiere examen pagado en partner AHLEI autorizado para el credential oficial. El comprobante Zenix Learning es interno alineado al estándar citado."*

### Módulo 1 — La industria hotelera y el rol de recepción (2h · 3 lessons)
- **L1.1** El error común: "recepción solo es dar llaves" *(20 min)*
- **L1.2** El ciclo del huésped (Kotler 4 fases) + tipología de establecimientos *(50 min)*
- **L1.3** Front-of-house vs back-of-house + el rol estratégico de recepción *(50 min)*

### Módulo 2 — Tecnología de recepción (3h · 3 lessons)
- **L2.1** El error común: pelear con el sistema en lugar de aprovecharlo *(30 min)*
- **L2.2** Qué es un PMS y cómo funciona (genérico): inventario, calendario, estados de habitación, folio *(75 min)*
- **L2.3** Channel managers + OTAs + GDS + el ecosistema de distribución hotelera *(75 min)*

### Módulo 3 — Pre-arrival y reservaciones (2h · 3 lessons)
- **L3.1** El error común: no confirmar proactivamente la reserva *(20 min)*
- **L3.2** Tipos de reserva (directa, OTA, GDS, walk-in) + garantías + depósitos *(50 min)*
- **L3.3** Overbooking, relocate policy y pre-arrival communication *(50 min)*

### Módulo 4 — Check-in excelente (3h · 4 lessons + 🎮 Game D capstone)
> **Capstone (Fase 1.5):** 🎮 Game D *"Front Desk Rush"* — time-management simulator de recepción (Diner Dash style). Ver [doc 23 §2](../../23-games-interactive-learning-strategy.md).
- **L4.1** El error común: check-in mecánico sin conexión humana *(30 min)*
- **L4.2** El proceso de check-in paso a paso: verificación ID, registro, asignación *(50 min)*
- **L4.3** Upselling ético en el check-in (Cialdini aplicado con integridad) *(40 min)*
- **L4.4** Casos especiales: VIP, grupos, walk-in, late arrival *(40 min)*

### Módulo 5 — Folio, cargos y pagos (3h · 3 lessons)
- **L5.1** El error común: leer en voz alta el número de tarjeta (violación PCI) *(30 min)*
- **L5.2** El folio + posting de cargos + métodos de pago + PCI-DSS básicos *(75 min)*
- **L5.3** OTA-collect vs hotel-collect + arqueo de caja + manejo de efectivo multi-divisa *(75 min)*

### Módulo 6 — Check-out y manejo de disputas (2h · 3 lessons)
- **L6.1** El error común: check-out apresurado que pierde la disputa después *(20 min)*
- **L6.2** Check-out + settle balance + early/late checkout + encuesta de salida *(50 min)*
- **L6.3** Disputa de cargos + chargebacks + evidencia Visa Core Rules *(50 min)*

### Módulo 7 — Servicio al huésped y manejo de quejas (2h · 3 lessons)
- **L7.1** El error común: ponerse defensivo ante una queja *(20 min)*
- **L7.2** El modelo LEARN (Listen, Empathize, Apologize, React, Notify) *(50 min)*
- **L7.3** Service recovery paradox + diferencias culturales LATAM *(50 min)*

### Módulo 8 — Seguridad, emergencias y protección de datos (2h · 3 lessons)
- **L8.1** El error común: revelar el número de habitación de un huésped *(20 min)*
- **L8.2** Seguridad física + emergencias + protocolo de evacuación *(50 min)*
- **L8.3** Protección de datos del huésped (GDPR/LFPDPPP) + registro federal *(50 min)*

### Módulo 9 — No-shows, fiscal y cierre de turno (1h · 2 lessons)
- **L9.1** El error común: no documentar el contacto previo al no-show *(25 min)*
- **L9.2** No-show policy + evidencia chargeback + night audit + cierre de turno *(35 min)*

## 3. Criterios de evaluación

### 3.1 Examen final
- **40 preguntas multiple choice** (selección aleatoria de banco ≥200, ratio 5×)
- **1 simulación de recepción:** check-in completo de un huésped ficticio en PMS genérico/conceptual (NO Zenix) — o Game D "Front Desk Rush" cuando esté disponible (Fase 1.5)
- **% aprobación:** 75% (alineado con CFDR oficial 70% + margen de calidad)
- **Re-takes:** 2 intentos con 48h espera; tercero requiere refresher de 4h

### 3.2 Module Quizzes
- Cada módulo cierra con 8-12 preguntas, aprobación ≥75%
- Drawn aleatoriamente del banco con Quiz Randomization Standard (doc 21 §1.1.bis)

### 3.3 Evidencia conservada
- Video/captura de simulación (data sintética, sin PII real)
- Score breakdown por módulo
- Registro de finalización para acreditar capacitación (CONOCER EC0124 alineado)

## 6. Errores comunes universales que el curso DEBE corregir

> Cada uno se incorpora como "El error común" del hook de la lesson correspondiente.

1. **"Recepción solo es dar llaves"** → M1 (rol estratégico: primera y última impresión, revenue, retención)
2. **"Pelear con el sistema"** → M2 (el PMS es aliado, no enemigo)
3. **"No confirmar la reserva proactivamente"** → M3 (pre-arrival communication reduce no-shows)
4. **"Check-in mecánico sin conexión humana"** → M4 (la primera impresión define toda la estadía)
5. **"Leer el número de tarjeta en voz alta"** → M5 (violación PCI-DSS + riesgo de fraude)
6. **"Asignar habitación sin verificar que está limpia"** → M4 (mover huésped a cuarto sucio)
7. **"Cobrar en USD al tipo de cambio inventado"** → M5 (FX inventado = problema fiscal + queja)
8. **"Check-out apresurado"** → M6 (disputa perdida después por falta de evidencia)
9. **"Ponerse defensivo ante queja"** → M7 (escala el conflicto en lugar de resolverlo)
10. **"Revelar el número de habitación de un huésped"** → M8 (riesgo de seguridad + violación de privacidad — caso documentado de stalking)
11. **"No registrar el contacto previo al no-show"** → M9 (no-show no defensable ante chargeback Visa 13.7)
